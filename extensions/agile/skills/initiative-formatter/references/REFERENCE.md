# initiative-formatter — reference

## What "strongly business-oriented" means

- **Include:** value, impact, who, why, observable outcomes.
- **Exclude:** implementation details, technology choices, architecture decisions — move to Notes if relevant.

## Success criteria guidance

- High-level and observable (not implementation steps).
- Concrete and achievable — avoid "best-in-class", "highly scalable", unbounded "fast".
- Prefer: "Users can complete checkout in under 3 steps" over "System is fast and user-friendly."
